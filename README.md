# Autonomous 3D Agent Streaming (A3AS)

## Technical Disclosure & System Architecture

This repository serves as a formal public disclosure of a decentralized system for anchoring autonomous 3D agents within a spatial environment (MapLibre).

## Core Innovation: Bitstream-to-glTF Mapping

The system replaces legacy "if-then" logic trees with a real-time bitstream that modifies glTF entities at the edge. Key features include:

### 1. Dynamic Mesh Modification

The bitstream directly modifies glTF morph targets and skinned meshes. This allows for procedural physical expressions driven by live data (e.g., environmental or inventory metadata) rather than static icon swapping.

### 2. Non-Deterministic Agent Inference

We utilize on-device LLM/LMM inference to drive agent behavior. This moves communication from "fixed notifications" to "generative interactions." The AI character possesses "motivations" (including advertising) rather than hard-coded triggers.

### 3. Distributed Spatial Logic

Using MapLibre custom layers, the system isolates the agent's intelligence from the map provider's central logic. Processing occurs locally (via WebAssembly or Edge API), bypassing centralized "Smart Assistant" patents.

### 4. Direct Communication & Contextual Ad-Streaming

Unlike current research into AI-driven 3D world building (which focuses on geometric analysis), this system prioritizes the "Interaction Layer."

- **Voice/Ad Popups/Video**: The streamed bitstream triggers specific emotive or vocal responses from the agent, effectively turning the glTF into an "interactive billboard" with its own AI persona.
- **Dynamic Interaction**: The agent's ability to trigger popups or voice interactions on the map is driven by real-time bits (e.g., IoT data from a fridge or local travel intent), rather than pre-baked world-building scripts.

### 5. Agent-to-Agent (A2A) Protocols

We disclose a protocol for **Multi-Agent Social Interaction**. Agents can interact with each other via:

- **Local Agentic Protocols**: Peer-to-peer communication between 3D entities on the client device.
- **LLM Negotiation**: Agents can "talk" to each other to coordinate map space, behaviors, or collaborative advertising.
- **Cross-App Communication**: Agents can respond to other AI agents or device-level protocols without central server intervention.

### 6. Gestural Agent Instantiation

The "long-press" gesture in our system is not utilized for static "pin dropping" (US8108144B2). Instead, it serves as a **Spatial Anchor Trigger** for the instantiation of a glTF AI agent.

- **Dynamic Initialization**: The gesture triggers a bitstream request to populate the coordinate with a unique 3D entity.
- **Non-Static Logic**: Unlike legacy map markers, these anchors are volatile and subject to Agent-to-Agent (A2A) negotiation protocols.
